## 1. 数据库表创建

### 执行 SQL 脚本

在数据库中执行以下 SQL 语句创建技能类型管理表：

```sql
-- 创建技能类型管理表
CREATE TABLE IF NOT EXISTS skill_type (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键 ID',
    type_code VARCHAR(50) NOT NULL UNIQUE COMMENT '类型代码：ATTACK, DEFENSE, SUPPORT, MOVEMENT, CULTIVATION',
    type_name VARCHAR(50) NOT NULL COMMENT '类型名称：攻击、防御、辅助、身法、功法',
    display_name VARCHAR(50) COMMENT '显示名称（前端标签显示）',
    description VARCHAR(500) COMMENT '类型描述',
    icon VARCHAR(100) COMMENT '图标',
    color VARCHAR(20) COMMENT '颜色代码',
    sort_order INT COMMENT '排序顺序',
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_type_code (type_code),
    INDEX idx_sort_order (sort_order),
    INDEX idx_is_active (is_active)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='技能类型管理表';

-- 初始化技能类型数据
INSERT INTO skill_type (type_code, type_name, display_name, description, icon, color, sort_order, is_active) VALUES
('SUPPORT', '辅助', '辅助', '辅助类技能，提供增益效果或特殊能力', '✨', '#9370DB', 1, TRUE),
('MOVEMENT', '身法', '身法', '身法类技能，提高移动速度和闪避率', '💨', '#20B2AA', 2, TRUE),
('ATTACK', '攻击', '攻击', '攻击类技能，造成直接伤害', '⚔️', '#DC143C', 3, TRUE),
('CULTIVATION', '功法', '功法', '修炼功法，提升整体实力', '📖', '#8B4513', 4, TRUE)
ON DUPLICATE KEY UPDATE 
    type_name = VALUES(type_name),
    display_name = VALUES(display_name),
    description = VALUES(description),
    icon = VALUES(icon),
    color = VALUES(color),
    sort_order = VALUES(sort_order);
```

### 验证表是否创建成功

```sql
-- 查看表结构
DESC skill_type;

-- 查看数据
SELECT * FROM skill_type ORDER BY sort_order;
```

## 4. 常见问题解决

### 4.1 找不到技能类型表

**错误信息：**
```
Table 'xxx.skill_type' doesn't exist
```

**解决方案：**
执行第 1 步的 SQL 脚本创建表

### 4.2 API 404 错误

**错误信息：**
```
GET http://localhost:8088/api/skill-type/active 404 (Not Found)
```

**解决方案：**
1. 检查后端服务是否启动
2. 确认 `SkillTypeController.java` 文件存在
3. 重启后端服务

### 4.3 getSkillTypes 方法不存在

**错误信息：**
```
TypeError: window.apiService.getSkillTypes is not a function
```

**解决方案：**
1. 清除浏览器缓存（见第 3.1 步）
2. 强制刷新页面
3. 检查 `js/api-service.js` 文件中是否有 `getSkillTypes` 方法（第 917 行）

### 4.4 前端没有显示技能类型标签

**可能原因：**
1. 后端 API 返回空数组
2. 前端缓存未清除

**解决方案：**
1. 检查数据库表是否有数据：`SELECT * FROM skill_type;`
2. 清除浏览器缓存并刷新
3. 查看控制台日志确认 API 调用是否成功

## 5. 后台管理操作

### 5.1 添加新的技能类型

**方式一：使用 SQL**
```sql
INSERT INTO skill_type (type_code, type_name, display_name, description, icon, color, sort_order, is_active) 
VALUES ('DEFENSE', '防御', '防御', '防御类技能，提高生存能力', '🛡️', '#4169E1', 5, TRUE);
```

**方式二：使用 API**
```bash
curl -X POST http://localhost:8088/api/skill-type \
  -H "Content-Type: application/json" \
  -d '{
    "typeCode": "DEFENSE",
    "typeName": "防御",
    "displayName": "防御",
    "description": "防御类技能，提高生存能力",
    "icon": "🛡️",
    "color": "#4169E1",
    "sortOrder": 5,
    "isActive": true
  }'
```

### 5.2 修改技能类型排序

```sql
-- 调整辅助类型为第 1 位
UPDATE skill_type SET sort_order = 1 WHERE type_code = 'SUPPORT';

-- 调整身法类型为第 2 位
UPDATE skill_type SET sort_order = 2 WHERE type_code = 'MOVEMENT';
```

### 5.3 启用/禁用技能类型

```sql
-- 禁用防御类型
UPDATE skill_type SET is_active = FALSE WHERE type_code = 'DEFENSE';

-- 启用防御类型
UPDATE skill_type SET is_active = TRUE WHERE type_code = 'DEFENSE';
```
