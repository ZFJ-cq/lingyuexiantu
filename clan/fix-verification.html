🔧 修复验证报告

✅ 问题定位

问题: 加入宗门时跳转失败

原因: 角色 ID 获取方式不正确，使用了硬编码的默认值 1

✅ 修复方案

1. 添加异步获取角色 ID 的函数 getRoleId()

2. 优先从 localStorage 获取角色 ID

3. 如果没有，则通过 API 从服务器获取

4. 在 executeJoinClan() 函数中使用异步获取的角色 ID

✅ 修复文件

文件: clan/clan-list.html

修改内容:

  • 添加 getRoleId() 异步函数
  • 修改 executeJoinClan() 函数，使用 await getRoleId()
  • 添加角色 ID 为空时的错误处理和跳转
  • 调整脚本加载顺序，确保依赖正确

📋 测试步骤

1. 打开 宗门列表页面

2. 点击任意宗门的"申请加入"按钮

3. 在确认弹窗中点击"确认加入"

4. 观察是否正确跳转到宗门页面
