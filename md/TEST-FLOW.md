# 灵月仙途 - 登录与角色同步测试流程

## 🎯 测试目标
确保登录后所有页面显示同一个角色

## 📋 测试步骤

### 第一步：清除所有旧数据
1. 打开浏览器开发者工具 (F12)
2. 点击 "Application" 标签
3. 展开 "Local Storage" → "http://localhost:8000"
4. 右键 → "Clear" 清除所有数据
5. 或者访问 http://localhost:8000/test-role-sync.html 点击"清除所有"按钮

### 第二步：测试登录 API
1. 访问 http://localhost:8000/test-login-api.html
2. 输入用户名和密码
3. 点击"测试登录"
4. 检查返回数据：
   - ✅ token: 必须存在
   - ✅ userId: 必须存在
   - ✅ roles: 角色列表
   - ⚠️ currentRoleId: 可能有也可能没有（正常）

### 第三步：正常登录
1. 访问 http://localhost:8000/login.html
2. 输入用户名和密码
3. 点击登录
4. 观察控制台日志：
   ```
   ✅ TokenManager: Token 已保存并备份
   📦 TokenManager: Token 已备份
   角色列表已保存：X 个角色
   ✅ 登录成功，跳转到角色选择页面...
   ```

### 第四步：选择角色
1. 自动跳转到 http://localhost:8000/start/start.html
2. 查看角色列表是否正确显示
3. 点击选择一个角色
4. 观察控制台日志：
   ```
   RoleSync: 设置角色 ID XXX
   ```
5. 点击"进入洞府"按钮

### 第五步：加载页面
1. 自动跳转到 http://localhost:8000/start/loading.html
2. 观察控制台日志：
   ```
   RoleSync: 统一使用角色 ID XXX
   RoleSync: 同步完成
   ```

### 第六步：检查首页角色
1. 自动跳转到 http://localhost:8000/index.html
2. 打开控制台 (F12)
3. 查看显示的角色名称
4. 在控制台输入：
   ```javascript
   console.log('首页角色 ID:', localStorage.getItem('currentRoleId'));
   ```

### 第七步：检查技能页面角色
1. 点击底部导航栏的"功法"按钮
2. 打开控制台 (F12)
3. 查看显示的角色名称
4. 在控制台输入：
   ```javascript
   console.log('技能页角色 ID:', localStorage.getItem('currentRoleId'));
   ```

### 第八步：验证角色一致性
1. 访问 http://localhost:8000/test-role-sync.html
2. 查看同步状态：
   - ✅ 所有角色 ID 已同步 → 通过
   - ❌ 角色 ID 不一致 → 失败

## 🔧 常见问题排查

### 问题 1：登录后直接跳转到加载页面，没有选择角色
**原因**：登录逻辑没有修改成功
**解决**：检查 login.html 第 316-320 行，应该是跳转到 start/start.html

### 问题 2：首页和技能页面角色不一致
**原因**：localStorage 中有旧的角色数据
**解决**：
1. 清除所有 localStorage 数据
2. 重新登录
3. 确保 RoleSync 已加载

### 问题 3：test-role-sync.html 显示角色 ID 不同步
**原因**：多个地方保存了不同的角色 ID
**解决**：
1. 点击"立即同步"按钮
2. 刷新所有页面
3. 重新检查
