# 🗄️ 数据库初始化说明

## 问题

数据库表已创建，但**没有测试数据**，导致无法登录。

## 解决方案

### 方案 1：通过 IDEA 执行初始化 SQL（推荐）

1. **打开 IDEA**
2. **打开 Database 面板**（右侧）
3. **连接到 lingyuexiantu 数据库**
4. **右键 → Run Data Script**
5. **选择文件**：
   ```
   /Users/macbook/前端项目/灵月仙途/lingyuexiantu-server/src/main/resources/data/insert-sample-data.sql
   ```
6. **执行**

### 方案 2：执行最小化初始化 SQL

如果只想快速创建测试账号，执行以下 SQL：

```sql
-- 创建测试账号 player001
-- 密码：123456 (BCrypt 加密)
INSERT INTO game_user (username, password, nickname, phone, status, created_at, updated_at)
VALUES ('player001', '$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iG6yL8xwC0hJQvK3yPz9sF4mNqW2', 
        '张三丰', '13900139001', 1, NOW(), NOW());

-- 创建角色
INSERT INTO game_role (user_id, role_name, gender, realm, level, hp, mp, spirit_root, status)
SELECT id, '张三丰', 1, 1, 1, 100, 100, 1, 1 
FROM game_user WHERE username = 'player001';
```

### 方案 3：使用完整初始化脚本

完整脚本位置：
```
/Users/macbook/前端项目/灵月仙途/lingyuexiantu-server/src/main/resources/data/insert-sample-data.sql
```

这个脚本会初始化：
- 系统角色（超级管理员等）
- 系统用户（admin 等）
- 游戏用户（player001-player005）
- 游戏角色
- 宗门数据
- 技能数据
- 装备数据
- 等等...

## 📱 初始化后测试

执行 SQL 后，在手机上：

1. **刷新登录页面**
2. **使用以下账号登录**：
   - 用户名：`player001`
   - 密码：`123456`
3. **应该成功登录**

## 🔍 检查是否已初始化

可以通过以下 SQL 检查：

```sql
-- 检查游戏用户
SELECT id, username, nickname FROM game_user;

-- 应该有 player001-player005 的记录
```

## ⚠️ 注意事项

1. **只需要执行一次**初始化脚本
2. **不要重复执行**，脚本中已有 `WHERE NOT EXISTS` 判断
3. **执行后重启后端**（可选，通常不需要）

---

**推荐**：使用 IDEA 执行 `insert-sample-data.sql` 完整初始化所有数据！
