# 灵月仙途 - 局域网访问配置指南

## 问题诊断

当你使用手机局域网访问项目时，如果只有前端展示但没有数据，通常是因为：

1. **后端服务未启动**
2. **跨域配置未允许局域网访问**
3. **前端 API 地址配置不正确**

## 已完成的修复

### 1. ✅ 后端 CORS 配置修复
已修改 `lingyuexiantu-server/src/main/java/com/lingyue/config/CorsConfig.java`，添加了对局域网访问的支持：
```java
// 允许所有局域网 IP 访问（开发环境）
config.addAllowedOriginPattern("*");
```

### 2. ✅ 前端环境检测修复
已修改 `js/config.js`，增加了局域网 IP 检测：
- 自动识别 `192.168.x.x`、`10.x.x.x`、`172.16-31.x.x` 网段
- 自动使用正确的 API 地址

## 启动步骤

### 方法一：使用启动脚本（推荐）

```bash
cd /Users/macbook/前端项目/灵月仙途
./start-backend.sh
```

脚本会自动：
- 检测系统环境（Java、Maven、MySQL）
- 显示你的局域网 IP 地址
- 提供三种启动方式供选择

### 方法二：手动启动

#### 1. 使用 Maven 启动
```bash
cd /Users/macbook/前端项目/灵月仙途/lingyuexiantu-server
mvn spring-boot:run
```

#### 2. 打包后运行
```bash
cd /Users/macbook/前端项目/灵月仙途/lingyuexiantu-server
mvn clean package -DskipTests
java -jar target/lingyuexiantu-server-*.jar
```

#### 3. 使用 IDEA 启动
- 打开 `lingyuexiantu-server` 项目
- 找到主类 `com.lingyue.LingyuexiantuServerApplication`
- 右键 → Run

## 验证服务启动成功

### 1. 检查端口监听
```bash
lsof -i :8088
```

应该看到类似输出：
```
COMMAND   PID   USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
java     1234  user   50u  IPv6  12345      0t0  TCP *:8088 (LISTEN)
```

### 2. 测试本地访问
```bash
curl http://localhost:8088/api/health
```

应该返回：
```json
{"code":200,"message":"success","data":"OK"}
```

### 3. 测试局域网访问

首先获取你的局域网 IP：
```bash
ipconfig getifaddr en0
# 或
ipconfig getifaddr en1
```

假设获取到的 IP 是 `192.168.1.100`，然后测试：
```bash
curl http://192.168.1.100:8088/api/health
```

## 手机访问方法

### 1. 确保手机和电脑在同一局域网

连接同一个 WiFi 网络

### 2. 获取电脑的局域网 IP

在电脑上执行：
```bash
ipconfig getifaddr en0
```

或在启动脚本中会自动显示

### 3. 手机访问地址

假设电脑 IP 是 `192.168.1.100`，前端运行在 `8080` 端口：

```
http://192.168.1.100:8080
```

### 4. 前端会自动配置 API 地址

当前端检测到是局域网访问时，会自动将 API 地址设置为：
```
http://192.168.1.100:8088/api
```

## 常见问题排查

### 问题 1：后端服务无法启动

**症状**：启动脚本报错或 Maven 命令找不到

**解决方案**：
1. 安装 Maven：`brew install maven`
2. 安装 JDK：`brew install openjdk@17`
3. 使用 IDEA 直接运行

### 问题 2：端口被占用

**症状**：`Port 8088 is already in use`

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :8088

# 杀死占用端口的进程
kill -9 <PID>

# 或者修改 application.yml 中的端口号
```

### 问题 3：数据库连接失败

**症状**：`Could not open JDBC Connection for transaction`

**解决方案**：
1. 确保 MySQL 服务已启动：`brew services list | grep mysql`
2. 启动 MySQL：`brew services start mysql`
3. 检查数据库配置：`lingyuexiantu-server/src/main/resources/application.yml`
4. 确认数据库已创建：
   ```sql
   CREATE DATABASE IF NOT EXISTS lingyuexiantu;
   ```

### 问题 4：手机访问仍然没有数据

**排查步骤**：

1. **检查防火墙**
   ```bash
   # macOS 系统偏好设置 → 安全性与隐私 → 防火墙
   # 确保允许 Java 或 8088 端口通过
   ```

2. **检查网络连通性**
   在手机上 ping 电脑 IP，确保能连通

3. **检查 API 地址**
   在手机浏览器打开开发者工具（如果有），查看网络请求
   - API 地址应该是 `http://你的IP:8088/api/xxx`
   - 如果是 `http://localhost:8088/api/xxx`，说明配置未生效

4. **清除浏览器缓存**
   在手机浏览器中清除缓存和 Cookie

5. **检查后端日志**
   ```bash
   tail -f lingyuexiantu-server/logs/lingyuexiantu.log
   ```

### 问题 5：跨域错误

**症状**：控制台显示 CORS 错误

**解决方案**：
1. 确认已修改 `CorsConfig.java` 并重启后端
2. 检查是否有多个 CORS 配置冲突
3. 确保 `application.yml` 中的 CORS 配置正确
