# 创建角色初始化数据完整清单

## 📋 概述

本文档详细说明了在《灵月仙途》游戏中，创建一个新角色时需要初始化的所有数据，确保前端所有页面能够正确显示。

---

## 🎯 核心初始化流程

### 1. 角色基础数据（GameRoleServiceImpl.createRole）

**表**: `game_role`

**初始化字段**:
```java
- realm: "无修为" (默认境界)
- level: 1 (默认等级)
- hp: 100 (默认气血)
- mp: 80 (默认法力)
- status: 1 (启用状态)
- createTime: 当前时间
```

**关联初始化**:
- 角色基础属性（role_base_stats）
- 角色资产（role_asset）- 所有资产类型初始化为 0

---

### 2. 角色基础属性（RoleStatsService）

**表**: `role_base_stats`

**初始化数据**:
```sql
INSERT INTO role_base_stats (role_id, vit, spi, agi, wis, lck)
VALUES (?, 10, 10, 10, 10, 10);
```

**属性说明**:
- vit: 根骨 (影响气血、防御)
- spi: 灵力 (影响法力、攻击)
- agi: 身法 (影响速度、闪避)
- wis: 悟性 (影响修炼速度)
- lck: 气运 (影响暴击、掉落)

---

### 3. 修炼配置（RoleDataService.initializeCultivationConfig）

**表**: `role_auto_cultivation_config`

**初始化数据**:
```sql
INSERT INTO role_auto_cultivation_config 
(role_id, is_enabled, cultivation_interval, auto_breakthrough)
VALUES (?, 0, 60, 0);
```

**字段说明**:
- is_enabled: 0 (关闭自动修炼)
- cultivation_interval: 60 (修炼间隔 60 秒)
- auto_breakthrough: 0 (关闭自动突破)

---

### 4. 境界数据

**表**: `role_realms`

**初始化数据**:
```sql
INSERT INTO role_realms (role_id, realm, xiuwei, level)
VALUES (?, '凡人', 0, 1);
```

**字段说明**:
- realm: 凡人 (初始境界)
- xiuwei: 0 (初始修为)
- level: 1 (境界等级)

---

### 5. 锻体数据（RoleDataService.initializeBodyCultivation）

#### 5.1 锻体境界

**表**: `role_body_cultivation`

**初始化数据**:
```sql
INSERT INTO role_body_cultivation 
(role_id, realm_id, body_exp, pain_value, tolerance, status)
VALUES (?, 1, 0, 0, 0, 1);
```

**字段说明**:
- realm_id: 1 (锻体初期)
- body_exp: 0 (锻体经验)
- pain_value: 0 (疼痛值)
- tolerance: 0 (耐受度)
- status: 1 (启用)

#### 5.2 身体部位进度

**表**: `role_body_part_progress`

**初始化数据**:
```sql
INSERT INTO role_body_part_progress 
(role_id, body_part_id, progress, is_locked)
SELECT ?, id, 0, 0 FROM body_part;
```

**说明**: 初始化 10 个身体部位的修炼进度

---

### 6. 任务数据（RoleDataService.initializeTasks）

**表**: `role_task`

**初始化数据**:
```sql
INSERT INTO role_task 
(role_id, task_id, status, progress, is_daily_task, accept_time)
SELECT ?, id, 'ACCEPTED', 0, 1, NOW() 
FROM task 
WHERE is_daily = 1 AND is_enabled = 1;
```

**说明**: 自动接取所有启用的日常任务

---

### 7. 新手物品（RoleDataService.giveStarterItems）

**表**: `role_item`

**初始化数据**:
```sql
-- 聚气丹 ×10
INSERT INTO role_item (role_id, item_id, count)
SELECT ?, id, 10 FROM item WHERE name = '聚气丹';

-- 回春丹 ×10
INSERT INTO role_item (role_id, item_id, count)
SELECT ?, id, 10 FROM item WHERE name = '回春丹';

-- 回灵丹 ×10
INSERT INTO role_item (role_id, item_id, count)
SELECT ?, id, 10 FROM item WHERE name = '回灵丹';
```

---

### 8. 新手礼包邮件（GameRoleServiceImpl.sendNewbieGift）

**表**: `mail` 和 `mail_item`

**默认礼包内容**:
- 灵石 ×1000
- 宗门贡献 ×500
- 云游点数 ×100
- 装备（随机武器、衣服、饰品各 1 件）
- 其他道具

**说明**: 通过邮件系统发放，需要手动领取

## 🐛 常见问题

### Q1: 创建角色后主页显示"加载中"卡住

**原因**: 资产数据未初始化
**解决**: 确保 `role_asset` 表有数据

### Q2: 修炼页面报错

**原因**: 修炼配置或境界数据缺失
**解决**: 检查 `role_auto_cultivation_config` 和 `role_realms` 表

### Q3: 锻体页面空白

**原因**: 锻体数据或身体部位进度缺失
**解决**: 检查 `role_body_cultivation` 和 `role_body_part_progress` 表

### Q4: 任务页面显示"暂无数据"

**原因**: 任务数据未初始化或 task 表没有日常任务
**解决**: 
1. 检查 task 表是否有 `is_daily=1` 且 `is_enabled=1` 的任务
2. 检查 `role_task` 表是否有记录

## 🎯 最佳实践

### 1. 事务处理

所有初始化操作应在一个事务中完成，确保数据一致性：

```java
@Transactional
public GameRole createRole(GameRole role) {
    // 保存角色
    GameRole savedRole = roleRepository.save(role);
    
    // 初始化所有数据
    roleDataService.initializeRoleData(savedRole.getId());
    
    return savedRole;
}
```

### 2. 错误处理

初始化失败不应阻止角色创建，但应记录日志：

```java
try {
    roleStatsService.calculateInitialBaseStats(savedRole);
} catch (Exception e) {
    e.printStackTrace();
    System.err.println("角色基础属性初始化失败：" + e.getMessage());
    // 继续执行其他初始化
}
```

### 3. 幂等性

所有初始化操作应支持重复执行（使用 `ON DUPLICATE KEY UPDATE`）：

```sql
INSERT INTO role_xxx (role_id, ...)
VALUES (?, ...)
ON DUPLICATE KEY UPDATE role_id = VALUES(role_id);
```
