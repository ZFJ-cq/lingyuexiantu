# 灵月仙途 - 开场动画使用说明

## 📱 页面概述

`intro.html` 是一个专为竖屏手机界面（9:16 比例）设计的文字修仙游戏开场动画页面，采用纯 CSS 和 JavaScript 实现，无需额外依赖。

## 🎞️ 动画结构

- 孤独书生剪影
- 银色残月坠落

## ✨ 核心功能

### 1. 动态水墨效果
- **水墨粒子**：每幕场景都有漂浮的水墨粒子
- **金墨滴落**：第一幕特有的金墨下落效果
- **墨水转场**：场景切换时使用墨水晕染效果

### 2. 文字动画
- **竖排文字**：第一幕采用传统竖排，从右到左阅读
- **逐行显示**：第二、三幕文字逐行淡入
- **高亮强调**：关键句子使用金色高亮

### 3. 交互功能
- **跳过按钮**：右上角随时可跳过动画
- **音频控制**：左上角音频开关（需添加音频文件）
- **进度条**：底部显示动画播放进度
- **开始按钮**：第三幕后点击进入游戏

### 4. 响应式设计
- **竖屏优化**：9:16 比例适配
- **横屏提示**：检测到横屏时显示提示
- **移动端适配**：支持不同尺寸的手机屏幕

## 🎨 视觉风格

### 配色方案
```css
--gold-primary: #e6c749;    /* 金色主色调 */
--gold-light: #f4d03f;      /* 亮金色 */
--gold-dim: #8a7525;        /* 暗金色 */
--text-main: #fdfbf7;       /* 主文字色 */
--text-dim: #d4c5a9;        /* 次要文字色 */
--bg-dark: rgba(15, 20, 35, 0.95);  /* 深色背景 */
```

### 动画效果
- **粒子浮动**：60 秒循环的背景粒子
- **墨水下落**：模拟雨水、瀑布的垂直运动
- **视差滚动**：背景与前景的不同速度移动
- **渐变晕染**：水墨扩散的视觉效果

## 🔧 自定义配置

### 修改文案
在 HTML 中找到对应的场景 div，修改文字内容：

```html
<!-- 第一幕竖排文字 -->
<div class="vertical-text">
  <span class="text-column">你的文案</span>
</div>

<!-- 第二幕横排文字 -->
<div class="horizontal-text">
  <div class="text-line">你的文案</div>
</div>

<!-- 第三幕文字 -->
<div class="final-text">
  <p>你的文案</p>
</div>
```

### 调整动画时长
在 JavaScript 的 `playIntro()` 函数中修改时间：

```javascript
// 第一幕持续时间（毫秒）
setTimeout(() => {
  switchScene(1, 2);
}, 14000);  // 改为你的时间

// 第二幕持续时间
setTimeout(() => {
  switchScene(2, 3);
}, 26000);  // 改为你的时间
```

### 添加背景音乐
1. 准备 MP3 格式的音频文件
2. 修改 audio 标签的 src 属性：
```html
<audio id="bgmAudio" loop>
  <source src="path/to/your/bgm.mp3" type="audio/mp3">
</audio>
```

### 调整粒子数量
在 `playIntro()` 函数中：
```javascript
createInkParticles('inkParticles1', 15);  // 粒子数量
createInkDrops('inkParticles1', 10);      // 墨滴数量
```

## 🚀 使用方式

### 方式一：直接访问
```bash
# 启动本地服务器
python3 -m http.server 8080

# 访问开场动画
http://localhost:8080/intro.html
```

### 方式二：作为游戏入口
将游戏的默认入口改为 `intro.html`，用户观看完动画后自动跳转到主页面。

### 方式三：嵌入现有流程
在登录成功后跳转到开场动画：
```javascript
// 登录成功后
window.location.href = 'intro.html';
```

## 🎯 性能优化建议

1. **粒子数量**：低端机型减少粒子数量（15→8）
2. **动画复杂度**：减少 CSS 滤镜使用（blur 等）
3. **预渲染**：如需更复杂效果，可考虑预渲染为视频
4. **音频编码**：使用 H.265 (HEVC) 编码减小包体

## 📝 注意事项

1. **音频自动播放限制**：现代浏览器需要用户交互后才能播放音频，默认静音
2. **状态栏遮挡**：重要视觉元素已下移，避开手机状态栏
3. **拇指热区**：交互按钮位于屏幕下 1/3，方便单手操作
4. **网络加载**：如需加载外部资源，建议添加加载动画

## 🎵 音频建议

推荐使用以下类型的古风音乐：
- **开场**：低沉、神秘的氛围音乐
- **中段**：逐渐增强的悬疑感
- **高潮**：史诗感的交响乐
- **结尾**：清脆的铃音或钟声

推荐音频格式：MP3 (128kbps) 或 AAC (256kbps)
