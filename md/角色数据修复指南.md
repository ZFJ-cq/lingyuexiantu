# 角色数据修复指南

## 问题描述
角色创建后，`gender`（性别）和 `realm`（境界）字段为 null，导致：
- 修炼页面无法显示境界信息
- 角色数据显示不完整

## 原因分析
1. `GameRole` 实体中 `gender` 是 `Integer` 类型（1=男，2=女），不是 `String`
2. 角色创建时没有设置默认性别
3. 旧数据可能缺少这些字段

## 解决方案

### 方法 1：使用修复接口（推荐）

在浏览器控制台执行以下 JavaScript 代码：

```javascript
// 修复角色 ID 为 1 的数据
fetch('http://localhost:8088/api/fix/role/1', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  }
})
.then(res => res.json())
.then(data => {
  console.log('修复结果:', data);
  if (data.code === 200) {
    console.log('✅ 修复成功！');
    console.log('角色数据:', data.data);
    // 刷新页面
    location.reload();
  } else {
    console.log('❌ 修复失败:', data.message);
  }
})
.catch(err => console.error('修复失败:', err));
```

### 方法 2：在 PHP 管理后台执行 SQL

如果您使用 PHP 管理后台，可以执行以下 SQL：

```sql
-- 修复角色性别和境界
UPDATE game_role 
SET 
    gender = 1,  -- 1=男，2=女
    realm = '炼气一层'
WHERE id = 1 AND (gender IS NULL OR realm IS NULL);
```

### 方法 3：重新创建角色

1. 删除现有角色
2. 在角色创建页面重新创建角色
3. 确保选择性别

## 验证修复

修复后，在浏览器控制台执行：

```javascript
// 获取角色数据
fetch('http://localhost:8088/api/role/1')
  .then(res => res.json())
  .then(data => {
    console.log('角色数据:', data);
    console.log('性别:', data.data?.gender);
    console.log('境界:', data.data?.realm);
  });
```

应该看到：
- `gender`: 1（男）或 2（女）
- `realm`: "炼气一层"

## 预防措施

### 修改角色创建逻辑

在 `GameRoleServiceImpl.java` 的 `createRole` 方法中添加：

```java
// 默认性别
if (role.getGender() == null) {
    role.setGender(1); // 默认男性
}
```

### 数据库约束

添加 NOT NULL 约束：

```sql
ALTER TABLE game_role 
MODIFY COLUMN gender TINYINT NOT NULL DEFAULT 1,
MODIFY COLUMN realm VARCHAR(50) NOT NULL DEFAULT '炼气一层';
```

## 常见问题

### Q: 修复后页面还是显示 null？
A: 清除浏览器缓存并刷新页面（Cmd+Shift+R）

### Q: 提示"角色不存在"？
A: 确认角色 ID 是否正确，当前默认是 1

### Q: 资源数据也不对？
A: 资源数据由后端自动初始化，重启后端即可

## 修复接口说明

**接口地址：** `POST /api/fix/role/{roleId}`

**功能：**
- 自动修复缺失的 gender 字段
- 自动修复缺失的 realm 字段
- 自动修复缺失的 level 字段

**返回值：**
```json
{
  "code": 200,
  "message": "修复成功",
  "data": {
    "id": 1,
    "roleName": "测试角色",
    "gender": 1,
    "realm": "炼气一层",
    "level": 1
  }
}
```
