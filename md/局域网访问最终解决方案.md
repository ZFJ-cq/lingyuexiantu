# 局域网访问最终解决方案

## 问题诊断

经过深入排查，发现问题的根本原因是：
- **Spring Security 的 CORS 配置没有正确生效**
- **GlobalCorsConfig 的 allowedOriginPatterns 不支持通配符匹配 IP 段**

## 临时解决方案（推荐）

### 方案 1：直接允许所有 Origin（仅开发环境）

修改 `/Users/macbook/前端项目/灵月仙途/lingyuexiantu-server/src/main/java/com/lingyue/config/SecurityConfig.java`：

```java
@Bean
public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration configuration = new CorsConfiguration();
    
    // 开发环境：允许所有 Origin
    configuration.addAllowedOriginPattern("*");
    configuration.setAllowCredentials(false); // 注意：通配符不能与 credentials 同时使用
    
    configuration.addAllowedMethod("GET");
    configuration.addAllowedMethod("POST");
    configuration.addAllowedMethod("PUT");
    configuration.addAllowedMethod("DELETE");
    configuration.addAllowedMethod("OPTIONS");
    configuration.addAllowedMethod("PATCH");
    
    configuration.addAllowedHeader("*");
    configuration.addExposedHeader("Authorization");
    configuration.setMaxAge(3600L);
    
    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", configuration);
    return source;
}
```

然后重启后端服务。

### 方案 2：使用具体的局域网 IP（更安全）

如果你知道确切的局域网 IP（如 192.168.110.98），直接添加：

```java
configuration.addAllowedOrigin("http://192.168.110.98:5502");
```

### 方案 3：禁用 Spring Security 的 CORS，只使用 GlobalCorsConfig

在 SecurityConfig.java 中：

```java
http
    .cors(cors -> cors.disable())  // 禁用 Security 的 CORS
    .csrf(csrf -> csrf.disable())
    // ... 其他配置
```

然后在 GlobalCorsConfig.java 中添加具体的 IP：

```java
.allowedOriginPatterns(
    "http://localhost:*", 
    "http://127.0.0.1:*",
    "http://192.168.110.98:5502"  // 直接写死你的 IP 和端口
)
```

## 快速修复步骤

1. 修改 SecurityConfig.java，使用方案 1 或方案 2
2. 重启后端服务：
   ```bash
   kill $(lsof -t -i:8088)
   cd /Users/macbook/前端项目/灵月仙途/lingyuexiantu-server
   java -jar target/lingyuexiantu-server-0.0.1-SNAPSHOT.jar
   ```
3. 在手机上访问：`http://192.168.110.98:5502`

## 测试

```bash
# 测试 CORS 是否生效
curl -X OPTIONS -H "Origin: http://192.168.110.98:5502" \
  -H "Access-Control-Request-Method: GET" \
  http://192.168.110.98:8088/api/health -v

# 应该看到 HTTP 200 和 Access-Control-Allow-Origin 头
```

## 为什么之前的配置不生效

1. **allowedOriginPattern 不支持 IP 段通配符** - Spring 的 CORS 配置对 IP 段的通配符支持有限
2. **Security 优先级高于 MVC** - Spring Security 的 Filter 在 MVC 之前执行，GlobalCorsConfig 被覆盖
3. **Credentials 与通配符冲突** - `setAllowCredentials(true)` 和 `allowedOriginPattern("*")` 不能同时使用

## 最终建议

**开发环境**：使用方案 1，允许所有 Origin（最简单）
**生产环境**：配置具体的域名和 IP，并启用 HTTPS
