**有问题的 API:**
- ❌ `GET /api/friend/list/{userId}` - 被当作静态资源
- ❌ `GET /api/inventory/{roleId}` - 被当作静态资源

### 4. 问题原因

新创建的控制器（FriendController, InventoryController）被 Spring Boot 的静态资源处理器拦截了。

**根本原因：** `GlobalCorsConfig` 中的 `addResourceHandlers` 配置导致所有非 `/static/**` 路径都被当作静态资源处理。

### 5. 解决方案

由于问题比较复杂，建议前端暂时使用模拟数据测试 UI 功能。
